HTML

  • DOCTYPE decleration specifies what the browser can expect to find on the page.
  • After DOCTYPE the HTML opening tag is used, followed by the language for the webpage.
  • head opening element contains metadata for the application, metadata is typically invisible but contains accessibility, seach engines, and performance.
  • title element within the head element is necessary for search engines.
  • style element defines CSS elements for the website.
  • base element defines the base URL for a webpage.
  • meta element defines metadata like the character set, description, keywords, author, and viewport.
  • script element embeds client side scripts into the HTML document.
  • The h1-6 elements are headings, with 1 being the biggest. Only one h1 tag is permitted.
  • ul is unordered list, li are bullet points inside ul.
  • img element contains image info, the src attribute within it defines the location of an image, width and height can also be defined.
  • a element makes a link, href attribute points towards the url.

CSS

  • Doing a seperate CSS document with rules applied to classes, like cards get a shadow, works to easily insert it.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Colors can be assigned by hex or by their name.
  • CSS links go below the last metadata but before the title.

Git

  • When working on individual features/problems create new branches to work in. Pull requests are used to merge comparison branches into the main branch. git checkout -b name-of-branch makesa new branch
  • git command precedes all git statements, checkout moves working branch to a new branch, -b flag makes a new branch.
  • git add adds modifications to our working branch to a staging area, if immediately followed by an -A tag it indicates that we want to add all changes.
  • git commit takes a snapshot of currently saves changes and your branch will be updated.
  • git pull receives your branches new modifications, origin specifies that these changes will come from github, and main indicates the branch.
  • git push origin followed by your branch name pushes your locally made changes to github.

JavaScript

  • Main purpose is to make the webpage functional using conditional logic.
  • Variables are containers that store values.
  • console.log can be used to see outputs after inspecting the element of your webpage.
  • Functions work through variables to see the value, variable names must match.
  • Take the time to review the GitHub tutorials on data types and how to call functions that were linked in prework module to further understanding, use all in a practice project if time permits until mastery is achieved.
